# 픽셀 씬 이미지 파이프라인 전환 — 설계 (2026-07-03)

## 목표

현재 코드로 그리는 320×180 픽셀 씬 4종(아파트/부모님 집/SOC/승강장)을
**codex CLI imagegen으로 생성한 고해상도 픽셀아트 배경 + 코드 제작 동적 이펙트**
구조로 교체한다. 기존 렌더 품질 문제(과도한 어둠, 광원뿔 경계, 노이즈 픽셀,
핫스팟 라벨 가림, 형태 어색함)를 이 전환으로 해소한다.

## 확정된 결정

| 항목 | 결정 |
| --- | --- |
| 배경 제작 | codex CLI(`codex exec`) imagegen — 프롬프트 파일을 읽어 생성 |
| 아트 스타일 | 고해상도 픽셀아트 (게임 픽셀 아이덴티티 유지) |
| 타깃 해상도 | 640×360 (현재 320×180의 정수 2배) |
| 동적/상호작용 오브젝트 | 코드 스프라이트 + 이펙트 (Phaser, 데이터 정의) |
| 기존 코드 아트 | 4장 전부 확정될 때까지 폴백 유지 → 확정 후 삭제 |

## 아키텍처

### 1. 프롬프트 팩 — `docs/art-prompts/`

씬별 1파일: `apartment.md`, `parents.md`, `soc.md`, `station.md`.
각 파일은 codex 에이전트에게 주는 **완결된 작업 지시서**:

- 공통 스타일 블록(각 파일에 중복 포함 — codex는 파일 하나만 읽음):
  사이버 누아르, 다크 블루/틸 기반 팔레트 + 씬별 액센트, 픽셀아트,
  16:9, 생성 크기 1280×720 이상
- 구도 명세: 기존 핫스팟 % 좌표에 맞춘 오브젝트 배치
  (예: 아파트 — 창문 우측 l69~98%, 워크스테이션 중앙 l35~55%)
- 품질 조항(기존 문제 예방): 하단 1/3 정보량 확보, 광원은 부드러운 감쇠,
  실루엣이 배경과 구분되는 명도 대비, 맥락 없는 낱개 픽셀 금지
- 산출 지시: imagegen 도구로 생성 후 `public/scenes/raw/<scene>.png` 저장

실행 방법 (Claude가 직접 수행, 실패 시 프롬프트 조정 후 재시도):

```powershell
Get-Content docs/art-prompts/apartment.md -Raw | codex exec -
```

### 2. 이미지 처리 — 다운스케일 파이프라인

- 원본: `public/scenes/raw/<scene>.png` (≥1280×720)
  (경로 변경: 원본은 assets-src/scenes-raw/ 로 이동 — dist 배포 크기 절감)
- 확정본: `public/scenes/<scene>.png` (640×360)
- PowerShell + .NET System.Drawing 최근접 이웃 리사이즈 스크립트
  (`scripts/downscale.ps1`) — 새 npm 의존성 없음
- 각 확정본은 실제 렌더 스크린샷으로 검수

### 3. 렌더 파이프라인 — `PixelScene` / `PhaserHost`

- Phaser 캔버스 320×180 → 640×360, `pixelArt` 유지
- 배경 PNG를 `preload`로 로드, 바닥 레이어로 표시
- 로드 실패/파일 없음 → 기존 코드 페인터로 자동 폴백 (전환기 안전망)
- 그 위에 씬별 **이펙트 페인터**(코드)가 동적 요소만 그림:
  - 아파트: 창밖 비, 모니터 글로우 펄스, 램프 플리커
  - 부모님 집: 창밖 비, TV 화면 애니메이션
  - SOC: 상황판 틱커/차트 갱신, 형광등 플리커, 커피머신 스팀
  - 승강장: 전광판 텍스트 스크롤, CCTV LED 점멸, 터널 헤드라이트, 비산 먼지
- 이펙트 좌표는 640×360 기준, 씬별 데이터 정의로 분리

### 4. 핫스팟 UX (이미지와 무관하게 즉시 적용)

- 라벨 상시 표시 → hover/focus 시에만 박스 하단 바깥에 표시
- 펄스 링(진행 유도)은 유지
- 이미지 통합 후 씬별 핫스팟 % 좌표 재보정 (렌더 스크린샷 기준)

### 5. 마무리 (4장 전부 확정 후)

- `src/game/phaser/art/` 페인터 모듈 삭제, 폴백 경로 제거
- README "코드로 그리는 원본 아트" → "원작 생성 이미지 배경 + 코드 이펙트" 갱신
- `scenes.test.ts` 등 관련 테스트 갱신, 전체 테스트 통과 확인

## 진행 순서

1. 핫스팟 UX 수정 + 렌더 파이프라인(배경 로드 + 폴백) + 다운스케일 스크립트
2. 프롬프트 팩 4종 작성
3. 씬별 반복: codex 생성 → 검수(불합격 시 프롬프트 조정·재생성) → 다운스케일
   → 통합 → 핫스팟 재보정 → 이펙트 결선 → 스크린샷 검증
4. 4장 확정 → 코드 아트 삭제 → 문서·테스트 갱신 → 전체 검증

## 리스크와 대응

- **imagegen이 구도 명세를 어김** → 핫스팟 % 재보정으로 흡수,
  편차가 크면 해당 부분만 프롬프트 강화 후 재생성
- **씬 간 스타일 편차** → 공통 스타일 블록 동일 문구 유지,
  4장을 같은 모델/세션 계열로 생성
- **codex imagegen 미동작/미설정** → 1씬 파일럿으로 먼저 검증하고
  실패 시 사용자에게 codex 설정 확인 요청
- **픽셀아트가 아닌 결과물** → 다운스케일이 어느 정도 픽셀화하지만,
  프롬프트에 픽셀 그리드·클러스터 크기를 명시해 원천 방지
